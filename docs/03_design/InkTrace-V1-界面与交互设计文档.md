# InkTrace 界面与交互设计文档 (UI/UE)

更新时间：2026-04-27
适用阶段：当前版本

---

## 1. 设计目标与原则

基于《InkTrace_V1_需求规格说明书》，本系统的界面设计必须遵循以下核心原则：

1. **沉浸式创作优先**：最大化纯文本编辑区，弱化所有非核心控制元素。
2. **极简状态反馈**：用最少的视觉干扰（如小圆点、角落文字）传达系统状态，杜绝阻断式弹窗（除非遇到无法挽回的错误或冲突）。
3. **响应式与性能优先**：在大纲/章节侧边栏支撑 500+ 列表时，保持滚动和拖拽的丝滑体验。

---

## 2. 全局样式规范 (Design System)

- **主色调**：
  - 背景色：极简灰白 `#FAFAFA` (白天模式) / 深灰 `#1E1E1E` (夜间模式预留)。
  - 侧边栏/控制区背景：纯白 `#FFFFFF`。
  - 主文本色：深灰黑 `#333333`（提高长时间阅读舒适度）。
  - 次要文本色（状态/统计）：浅灰 `#999999`。
- **状态色**：
  - 成功/已保存：绿色 `#52C41A`。
  - 进行中/保存中：蓝色 `#1890FF` 或 灰色 Loading 动画。
  - 警告/离线/失败：橙红色 `#FA8C16`。
- **排版 (Typography)**：
  - 章节列表字体：`14px`，无衬线或系统默认 UI 字体。
  - 正文编辑字体：`16px` 或 `18px`，行高 `1.8`，推荐使用护眼的等宽字体或类似宋体/黑体的阅读字体。

## 3. 核心页面线框图 (Wireframes)

### 3.1 页面一：书架 (WorksList.vue)

作为进入系统的第一入口，界面需保持整洁，突出“新建”与“导入”核心动作。参考阅文作家助手的卡片式排版与顶部快捷操作栏。

```text
+-----------------------------------------------------------------------------+
|  [ InkTrace Logo ]                                            [ 设置 ] [ 退出 ] |
|-----------------------------------------------------------------------------|
|                                                                             |
|  +------------------------+  +------------------------+                     |
|  | 📄 新建                  |  | ⬇️ 导入                  |                     |
|  | 私密作品、阅文作品、分组 |  | 本地导入私密作品          |                     |
|  +------------------------+  +------------------------+                     |
|                                                                             |
|                                                               👁️ 已隐藏  🗑️ 回收站 |
|  +-------------------+  +-------------------+  +-------------------+        |
|  | 📖 星辰变           |  | 📖 凡人修仙传      |  | 📖 修仙从儿时开始    |        |
|  |                   |  |                   |  |                   |        |
|  |                   |  |                   |  |                   |        |
|  |                   |  |                   |  |                   |        |
|  +-------------------+  +-------------------+  +-------------------+        |
|  星辰变         (...)   凡人修仙传       (...)   修仙从儿时开始     (...)   |
|  280.4万字              15.2万字               1.9万字                      |
+-----------------------------------------------------------------------------+
```

**视觉与交互规范 (UI/UE)**：
- **顶部操作栏**：横向排列的大按钮（新建、导入），带有轻微阴影，悬浮时背景色微变。
- **作品卡片排版**：
  - 封面区域为主视觉（默认显示带有羽毛/笔图标的纯色背景，左上角可带状态标签如“私密”）。
  - 卡片下方独立显示作品名称和字数。
  - 名称右侧带有 `(...)` 更多操作按钮。
- **更多操作菜单**：点击 `(...)` 弹出下拉菜单，包含：
  - 导出 TXT
  - 重命名
  - 删除（点击后弹出二次确认框：“此操作不可恢复，确认删除？”）
- **跳转动作**：点击作品卡片封面或名称，直接跳转进入写作页（自动定位上次阅读的章节）。

#### 3.2.1 左侧：章节侧边栏 (ChapterSidebar)
**视觉设计**：
- 顶部固定 `[新建章节]` 按钮（大按钮，显著突出）。
- 列表支持过滤/分组（如“草稿”、“已发布”选项卡，V1可仅保留单列表）。
- 列表项高度紧凑，左侧显示标题，**右侧对齐显示字数**（如 `2122`）。
- 当前激活的章节背景色微变（如极浅的蓝色/灰色）。

**交互规范 (UE)**：
- **新增章节**：点击顶部 `[新建章节]` 或底部 `+ 新建章节`，在末尾或当前激活章节的下方插入一个“第X章”，并自动聚焦到中央编辑区。
- **Hover 行为规范 (防乱加功能)**：鼠标 Hover 到非激活的列表项上时，右侧字数隐藏，浮现极简操作图标（依次排列）：
  - `[铅笔]`：重命名章节（点击进入内联输入框，失去焦点或回车即保存）。
  - `[垃圾桶]`：删除章节（点击后**必须**弹出二次确认框）。
- **拖拽排序**：长按列表项即可拖拽排序，松手后触发接口。

#### 3.2.2 中央：沉浸式编辑区 (PureTextEditor)
**视觉设计**：
- 顶部为大号字体的标题输入框（如 `第142章 大陆，起！`）。
- 正文区域两侧留白极宽（类似 Word 页面居中），提升阅读体验，单行字数控制在最佳阅读区间。
- 底部状态栏显示字数统计（如 `本章: 2070`）。

**交互规范 (UE)**：
- **纯文本输入**：完全拦截富文本。支持顺滑的滚轮操作。
- **打字机模式 (可选)**：保持当前输入行始终位于屏幕垂直中央。
- **切换章节保护 UI (防丢字)**：
  - 若当前章节状态为 `已保存` 或 `保存失败(已写本地缓存)`：点击其他章节**无感切换**（瞬间切换），系统在后台静默保存当前会话的光标与滚动条位置。
  - 若当前章节状态为 `保存中`：点击其他章节时，**阻止切换动作**，在顶部短暂弹出轻提示（Toast）“正在同步数据，请稍候...”，等保存完成后自动执行切换。

#### 3.2.3 右侧：快捷工具栏 (预留)
**视觉设计**：
- 极窄的图标列（如宽度 `50px`）。
- 包含大纲、角色、设定等扩展入口。
- **【绝对约束】**：V1 阶段右侧栏**默认隐藏**，只允许用作图标占位。**绝对不允许**点击图标展开任何复杂面板或抽屉。这是为了防止系统重蹈“复杂 UI 导致崩溃”的覆辙。

#### 3.2.4 保存状态视觉规范 (State Machine UI)

顶部 Header 右侧的状态指示器需严格映射以下 UI 表现，禁止开发者自由发挥：

| 内部状态 | UI 表现 (图标 + 颜色 + 文本) | 交互说明 |
| :--- | :--- | :--- |
| **已保存** | 🟢 绿色对勾 + `已同步` | 静默显示，无动画干扰 |
| **保存中** | ⚪ 灰色 Loading 圈 + `保存中...` | 用户输入防抖期间显示 |
| **保存失败** | 🟠 橙色警告符 + `同步失败` | 提示后端异常，但**不阻断**本地继续输入 |
| **离线模式** | 🟡 顶部通栏极窄黄条 + `离线模式` | 断网时全局常驻提示：“当前网络离线，已开启本地缓存保护”，网络恢复后自动消失 |

## 4. 关键交互弹窗与反馈 (Modals & Toasts)

### 4.1 乐观锁冲突弹窗 (Version Conflict Modal)

当发生并发覆盖危险（API 返回 409）时，必须阻断用户继续提交，强制决策。

```text
+-------------------------------------------------+
| ⚠️ 检测到版本冲突                               |
|-------------------------------------------------|
| 您当前编辑的章节，在云端已经被其他设备或标签页  |
| 更新过。                                        |
|                                                 |
| 如果继续保存，将会覆盖云端的最新内容。          |
|                                                 |
|  [ 放弃本地修改，重新加载云端 ]  [ 强制覆盖云端 ] |
+-------------------------------------------------+
```

### 4.2 轻量级提示 (Toast)
用于非阻断式的信息反馈（显示 3 秒后消失）：
- “作品导出成功，开始下载” (绿色)
- “导入成功，共解析 120 章” (绿色)
- “网络已断开，已开启本地缓存模式” (橙色)

## 6. 特殊状态规范 (加载态 / 空态 / 错误态)

本节是防止页面“白屏”或“假死”的关键体验兜底。

### 6.1 书架页 (WorksList)
- **空态 (第一次打开)**：居中显示大型占位图（插画）+ 主文案“暂无作品” + 副文案“开始你的第一部伟大著作吧” + 大号 `[新建作品]` 按钮。
- **加载态**：列表区域显示骨架屏 (Skeleton) 或优雅的居中 Loading 动画。
- **错误态**：接口超时或 500 时，显示“获取作品列表失败” + `[点击重试]` 按钮。

### 6.2 写作页 (WritingStudio)
- **无章节空态 (极端情况)**：左侧章节列表为空，中央编辑区不可输入（或置灰），居中显示“请先新建章节” + `[+ 新建章节]` 按钮。
- **加载章节中**：点击切章或初次进入时，中央编辑区显示骨架屏（模拟几行文字的灰色占位条）。
- **加载失败**：中央编辑区显示“章节内容加载失败” + `[重新加载]` 按钮，不影响侧边栏的其他操作。

## 7. 遗留 UI 清理说明

基于新的极简纯文本设计理念，开发时需**彻底移除**以下 V1 遗留 UI 组件：
- 移除所有与 AI 对话、剧情弧线（Arc）相关的抽屉或浮窗。
- 移除富文本编辑器（TipTap）及其悬浮菜单（BubbleMenu）。
- 移除右侧复杂的“设置”、“知识库”、“人物卡”侧边栏。
