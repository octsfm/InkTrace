# InkTrace V1.1 需求规格说明书（非 AI 写作增强）

版本：v1.1  
更新时间：2026-04-29  
范围定位：在 V1 纯文本写作主链路可用的前提下，补齐“纯写小说”刚需能力，并加入结构化写作辅助模块（不包含任何 AI 能力）。  
V1.1 是 V2 AI 接入前的非 AI 创作工作台完整阶段，目标是完成写作体验收口与结构化写作资产沉淀。

---

## 1. 阶段关系

明确阶段关系：

- V1：稳定纯文本写作底座
- V1.1：非 AI 创作工作台（结构化写作资产沉淀）
- V2：基于结构化资产接入 AI 能力（仅在此阶段接入 AI）

---

## 2. 交付批次（V1.1）

- V1.1-A：写作体验与数据闭环收口（必须交付）
- V1.1-B：非 AI 写作组织能力（必须交付，但后置批次）
- V1.1-C：体验偏好增强（可选交付）

交付说明：

- V1.1-A 可单独提测与验收
- V1.1-B 完成后才视为 V1.1 完整完成（V1.1-C 是否交付不影响 V1.1 完整性的定义）

---

## 3. 目标与原则

### 3.1 目标（结果导向）

- 正文区更干净：长时间写作不被 UI 噪音打断
- 不丢稿：Local‑First 自动保存可靠、可恢复
- 可组织：大纲/细纲、时间线、伏笔、人物卡支撑长篇写作
- 可管理：作品信息、章节定位、导入导出闭环

### 3.2 产品原则

- 正文区优先：中间正文区域只呈现“标题输入框 + 正文 + 必要状态”
- Local‑First：输入立即写本地缓存；debounce(2–3s) 同步；成功清缓存
- 单容器承载：写作辅助模块统一使用一个右侧抽屉容器（Tab 切换），避免多面板堆叠

### 3.3 “非 AI”边界（写硬）

V1.1 明确禁止：

- 禁止生成正文（包括续写、改写、润色、扩写、缩写）
- 禁止自动分析正文（包括情节分析、人物抽取、伏笔抽取、时间线抽取）
- 禁止自动推荐剧情、自动推荐人物关系、自动补全设定
- 禁止根据正文自动创建/更新任何结构化资产（大纲/时间线/伏笔/人物卡）

所有内容均由用户手动创建、编辑、排序、关联。

## 4. 规则编号体系与书写格式

### 4.1 编号体系

- R-WS-XX：写作页体验
- R-WORK-XX：作品管理
- R-NAV-XX：搜索与定位
- R-IO-XX：导入导出
- R-OUT-XX：大纲
- R-TL-XX：时间线
- R-FS-XX：伏笔
- R-CHAR-XX：人物卡
- R-PREF-XX：偏好/专注模式

### 4.2 每条需求格式

每条需求统一写成：

- Rule：规则与意图
- Behavior：用户可观察行为
- Edge Case：边界与异常行为
- Acceptance：可验证验收点

## 5. V1.1-A（必须）：写作体验与数据闭环收口

### R-WS-01 正文区极简（正文不被挤压）

- Rule：中间正文区域必须极简，正文可视高度优先。
- Behavior：
  - 中间只保留：居中章节标题输入框 + 正文输入区 + 底部轻量信息（字数/纯文本提示/保存状态以不打扰方式呈现）。
  - 移除“说明/调试占位文案”（如会话已加载、光标、滚动等）。
- Edge Case：
  - 窗口高度较小时（< 700px），标题区高度必须进一步收缩，保证正文仍有足够高度可输入。
- Acceptance：
  - 进入写作页后，正文输入区在首屏可见区域占主要高度；不存在成块说明文字占位。

### R-WS-02 章节标题输入框（B2）+ 同次 flush 保存

- Rule：标题输入框显示“第X章 + 标题”，编辑后与正文共享 debounce，title+content 必须同次 flush 提交。
- Behavior：
  - 输入框展示：`第{order_index}章 {title}`（title 为空则仅展示 `第{order_index}章`）。
  - 用户编辑后，2–3 秒内自动保存（与正文同节流）。
  - 保存到后端只写入用户标题 title（可为空），不写入“第X章”前缀。
- Edge Case：
  - 标题为空：允许保存空 title；UI 只展示 `第{order_index}章`（不把 `第{order_index}章` 写入 title，避免出现 `第1章 第1章`）。
  - 用户手动删除“第X章”前缀：允许；保存时仍按“去前缀规则”提取 title；若最终为空则保存空 title。
  - 多次快速编辑：只提交最后一次编辑结果。
- Acceptance：
  - 连续编辑标题与正文，停止输入 ≤3 秒后刷新页面，标题与正文均不丢失，且两者更新时间一致（同次提交）。

### R-WS-03 Header 显示作品名（不显示 UUID）

- Rule：Header 只展示作品名（可选作者）与保存状态，不展示 workId。
- Behavior：
  - Header 主文案为作品名；作者为空则不显示作者行。
- Edge Case：
  - 作品名过长：单行省略号截断；hover/点击可查看完整名称（tooltip）。
- Acceptance：
  - 页面中不出现 UUID 作为面向用户的展示文案。

### R-WS-04 新建章节默认行为：追加到末尾（统一规则）

- Rule：V1.1 新建章节默认行为改为“追加到末尾”。插入当前章后能力若保留，必须作为显式二级操作（不作为默认）。
- Behavior：
  - 点击“+ 新章”，无论当前选中哪章，都追加到最后一章后。
  - 创建成功后：自动选中新章 + 章节列表滚动到新章 + 正文区聚焦。
- Edge Case：
  - 当前存在章节搜索过滤：创建新章后自动清空搜索并定位新章（避免“新章创建成功但列表不可见”）。
- Acceptance：
  - 在任意章节上点“+ 新章”，新章都出现在列表末尾并被选中。

### R-WS-05 章节编号显示（第X章 = order_index）

- Rule：章节编号必须可见且稳定，X 以 order_index 为唯一依据。
- Behavior：
  - 左侧列表每一项至少展示 `第{order_index}章`，标题为空也展示编号。
- Edge Case：
  - 若后端偶发返回缺失 order_index：前端仅兜底使用列表 index+1 展示，但不得写回后端。
- Acceptance：
  - 章节列表不存在“0章/无编号”展示。

### R-WORK-01 作品管理闭环（书架 `(...)`）

- Rule：作品管理必须闭环，书架卡片 `(...)` 提供必需操作。
- Behavior：
  - `(...)` 菜单包含：重命名作品、修改作者、删除（二次确认）。
- Edge Case：
  - 删除后自动刷新列表；若删除的是当前打开作品，返回书架并提示“作品已删除”。
- Acceptance：
  - 从书架完成“创建 → 改名/改作者 → 删除”全流程无死角入口。

### R-NAV-01 章节搜索（只过滤展示，不改变真实顺序）

- Rule：搜索只影响展示列表，不影响章节真实顺序与 order_index。
- Behavior：
  - 输入关键词后仅展示命中项，顺序仍按 order_index。
- Edge Case：
  - 搜索状态切章：允许；清空搜索后回到完整列表且当前章仍高亮。
- Acceptance：
  - 搜索不改变章节编号与顺序；退出搜索后顺序保持不变。

### R-NAV-02 跳转到第 N 章（N 从 1 开始）

- Rule：提供“跳转到第 N 章”能力，N 从 1 开始计数。
- Behavior：
  - 输入整数 N 并确认后，选中并打开对应章节。
- Edge Case：
  - 输入非数字：提示“请输入章节序号（数字）”。
  - N 超出范围：提示“章节序号超出范围（1–{max}）”。
- Acceptance：
  - 跳转后章节列表滚动到激活项，正文区显示对应内容。

### R-NAV-03 激活项可见（统一触发点）

- Rule：任何导致 active chapter 变化的场景，都必须保证激活项可见（scrollIntoView）。
- Behavior：触发场景包含：
  - 切章
  - 新建章节
  - 删除章节后选择下一章
  - 搜索跳转
  - 会话恢复/刷新后自动打开 last_open_chapter
- Edge Case：
  - 若列表正在拖拽中，延后滚动到拖拽结束。
- Acceptance：
  - 以上场景不会出现“active 已切换但列表停留在另一处”的情况。

### R-IO-01 Web TXT 导入（上传导入）

- Rule：Web 端导入必须使用文件选择器并上传文件内容，禁止依赖 file_path。
- Behavior：
  - 用户选择 `.txt` 文件后上传导入，导入完成创建作品并进入写作页。
- Edge Case：
  - 空文件：导入成功，创建作品与单章，内容为空。
  - 无章节标记：导入成功，创建单章并填入全文内容。
  - 编码：按 utf-8-sig → utf-8 → gb18030 顺序尝试；均失败则提示“文件编码无法识别，请转换为 UTF-8”。
  - 文件大小上限：20MB；超过提示“文件过大，请拆分后导入（上限 20MB）”。
  - 章节顺序校正：导入后 order_index 必须连续从 1..N。
- Acceptance：
  - Web 端可完成选择文件→导入→进入写作页；导入后章节顺序正确。

### R-IO-02 TXT 导出（格式选项）

- Rule：导出必须按 order_index 拼接，并支持常用格式选项。
- Behavior：
  - 导出顺序：order_index 升序。
  - 选项：
    - include_titles（是否包含章节标题）
    - gap_lines（章节之间空行 0/1/2）
- Edge Case：
  - 文件名规则：`{作品名}-{YYYYMMDD}.txt`，作品名做非法字符清理；作品名为空则用“未命名作品”。
  - 空作品：导出空文件（0 字节，UTF-8 无 BOM），仍视为成功。
  - 拼接规则：
    - include_titles=true：每章输出标题行 + 换行 + 正文
    - gap_lines=n：章与章之间追加 n 个空行（即 n 次 `\n`）
- Acceptance：
  - 导出文件顺序正确、选项生效、文件名符合规则。

## 6. V1.1-B（必须）：非 AI 写作组织能力（结构化资产沉淀）

### 6.0 保存策略（显式保存 + 乐观锁）

- Rule：V1.1-B 的结构化资产默认采用“显式保存”策略，避免将所有资料编辑都接入正文级 Local‑First 自动保存，降低复杂度。
- Behavior：
  - 抽屉内每个模块编辑区提供“保存”按钮（或 Ctrl/Cmd+S），点击后提交更新。
  - 更新请求携带 `expected_version`（乐观锁）；后端成功后 `version+1` 返回。
  - 保存成功以轻提示反馈；保存失败提供可恢复重试入口。
- Edge Case：
  - 409 冲突：弹窗提示“云端已更新”，提供“覆盖保存 / 放弃并刷新”。
  - 离线：允许继续编辑但不自动排队回放；离线暂存使用本地缓存，但不进入自动回放队列；显示“离线，未保存”状态；联网后用户手动点击保存触发提交。
- Acceptance：
  - 修改后用户能明确感知是否已保存；冲突不会静默覆盖；离线不丢失编辑内容（至少本地暂存）。

### 6.1 数据模型（最小字段，独立于旧 AI 语义）

本批次新增表不复用旧 AI 时代的世界观/剧情弧/角色体系，避免语义污染。

**work_outlines**
- id (uuid), work_id (uuid, unique), content_text (text), content_tree_json (json, nullable), version (int), created_at, updated_at

**chapter_outlines**
- id (uuid), chapter_id (uuid, unique), content_text (text), content_tree_json (json, nullable), version (int), created_at, updated_at

**timeline_events**
- id (uuid), work_id (uuid), order_index (int), title (text), description (text), chapter_id (uuid, nullable), created_at, updated_at

**foreshadows**
- id (uuid), work_id (uuid), status (text: open|resolved), title (text), description (text), introduced_chapter_id (uuid, nullable), resolved_chapter_id (uuid, nullable), created_at, updated_at

**characters**
- id (uuid), work_id (uuid), name (text), description (text), aliases (json array, nullable), created_at, updated_at

排序字段：
- timeline_events：order_index
- 其余列表默认按 updated_at desc（可后续调整）

### 6.2 关联与删除规则（安全优先）

删除章节时：
- chapter_outlines：随章节删除（一对一绑定章节语义，避免孤儿数据）
- timeline_events.chapter_id：置空（保留事件资料）
- foreshadows.introduced_chapter_id / resolved_chapter_id：置空（保留伏笔资料与状态）
- outline 节点若存在“关联章节”字段：置空（保留大纲资料）

删除作品时：
- 结构化资产（work_outlines / timeline_events / foreshadows / characters）随作品删除
- chapters 与其 chapter_outlines 随作品删除

### 6.3 右侧抽屉容器规则

- 同一时间只允许一个抽屉打开
- 抽屉 Tab：大纲、时间线、伏笔、人物
- 打开/关闭抽屉不主动改变正文焦点；用户主动点击抽屉内输入控件时允许焦点进入抽屉；关闭抽屉后不强制抢回焦点
- 切章时：
  - 章节细纲自动切换到对应章节
  - 全书大纲、时间线、人物卡保留当前筛选/滚动状态
- 移动端（≤768px）：
  - 抽屉变为全屏覆盖层（或底部覆盖层），关闭按钮必须明显

### R-OUT-01 大纲（Outline）与细纲（Chapter Outline）

- Rule：提供“全书大纲 + 本章细纲”两级结构，均为用户手写资产；支持保存与版本冲突处理。
- Behavior：
  - 全书大纲：在“大纲”Tab 中编辑；保存后全作品可见。
  - 章节细纲：随切章自动切换到当前章节；保存仅影响该章节细纲。
  - 树状视图：轻树（≤3 层），支持节点增删改；树数据与文本数据可并存，文本为兜底。
- Edge Case：
  - 章节被删除：该章节的细纲随章节删除；全书大纲不受影响。
  - 树状数据损坏：不影响文本展示与保存（文本仍可编辑）。
- Acceptance：
  - 切章后细纲跟随切换；保存后刷新页面内容仍在；冲突时提示并要求决策。

### R-TL-01 时间线（Timeline，顺序型）

- Rule：时间线是顺序事件列表，不引入真实时间；支持排序与章节关联跳转。
- Behavior：
  - 事件字段：title、description、order_index、可选 chapter_id。
  - 支持新增/编辑/删除事件；支持拖拽或上移/下移改变顺序；保存后顺序稳定。
  - 排序保存时一次性提交受影响事件的 `[{ id, order_index }]` 列表，后端单事务写入。
  - 若事件关联章节，点击事件可跳转到对应章节。
- Edge Case：
  - 关联章节被删除：chapter_id 置空，事件保留。
- Acceptance：
  - 顺序调整后刷新仍保持；关联章节跳转准确；删除章节不会误删事件。

### R-FS-01 伏笔/线索（Foreshadow）

- Rule：伏笔是可追踪资产，支持状态（未回收/已回收）与章节引用；默认以“未回收”视角工作。
- Behavior：
  - 记录字段：title、description、status(open/resolved)，可选 introduced_chapter_id/resolved_chapter_id。
  - 列表默认展示 open；可切换查看 resolved；支持一键切换状态。
  - 若关联章节存在，点击引用可跳转到章节。
- Edge Case：
  - 引入/回收章节被删除：对应引用置空，伏笔记录保留。
- Acceptance：
  - 未回收列表可用；状态切换可用；引用失效不会导致记录丢失。

### R-CHAR-01 人物卡（Character）

- Rule：人物卡仅提供最小可用信息沉淀（name/description/aliases），支持搜索，避免滑向复杂关系网。
- Behavior：
  - 支持新增/编辑/删除人物卡；aliases 为 JSON array（前端可用逗号输入转换）。
  - 搜索按 name 与 aliases 匹配（大小写不敏感）。
- Edge Case：
  - name 重名：允许但提示（不强制唯一）。
- Acceptance：
  - 人物卡 CRUD 与搜索可用；刷新后仍在；不依赖 AI。

## 7. V1.1-C（可选）：体验偏好增强

### R-PREF-01 全屏/专注模式

- Rule：提供一键专注模式，隐藏侧边栏与 Header，仅保留状态条与正文。
- Behavior：进入专注模式后正文区占满主要空间；退出后恢复原布局与滚动位置。
- Edge Case：专注模式下仍可打开右侧抽屉（覆盖层方式），关闭后回到正文。
- Acceptance：专注模式切换不丢稿、不重置光标滚动。

### R-PREF-02 书写偏好（本地持久化）

- Rule：字体/字号/行距/主题（浅/深）可配置并本地持久化。
- Behavior：刷新页面偏好仍生效。
- Edge Case：无可用字体时 fallback 到系统字体。
- Acceptance：偏好可见且可恢复。

### R-PREF-03 今日新增字数口径（写死）

- Rule：只统计正文有效字符新增，不含标题；以本地编辑行为为准；日期按本地时区自然日；净新增最低为 0。
- Behavior：当日新增随输入增长；删除文字导致净增为负时显示 0。
- Edge Case：跨日写作自动切换统计日。
- Acceptance：当日新增在刷新后仍保持一致（本地存储）。

### R-PREF-04 立即同步按钮（可选交付）

- Rule：提供“立即保存/立即同步”入口提升心理安全感，不改变自动保存机制。
- Behavior：点击后触发一次 flush，将当前待同步内容立刻提交。
- Edge Case：离线或服务异常时提示失败并允许重试。
- Acceptance：点击后能观察到立即发起同步请求；成功后状态变为“已同步”。

## 8. DoD（分批验收）

### 8.1 V1.1-A DoD（写作体验与数据闭环）

- [ ] 正文区极简：标题输入框 + 正文 + 底部轻量信息；无说明/调试占位文案；正文不被 Header/标题区挤压
- [ ] 标题输入框（B2）与正文同次 flush 保存：停 3 秒内完成自动保存；刷新后不丢；不出现 `第1章 第1章` 重复

## 9. 验收矩阵（测试维度）

按模块覆盖：正常路径 / 边界路径 / 异常路径 / 数据持久化 / 刷新与重启恢复。

**写作页体验（WS）**
- 正常：创建/切章/输入/保存
- 边界：搜索状态新建/跳转、极小窗口高度
- 异常：断网写作→联网回放、409 冲突覆盖/放弃
- 持久化：刷新不丢标题与正文，光标/滚动恢复

**作品管理（WORK）**
- 正常：新建→改名/改作者→删除
- 异常：删除失败提示与可重试
- 恢复：删除后刷新书架数据一致

**导航（NAV）**
- 正常：搜索、跳转第 N 章
- 边界：非数字/越界输入提示
- 恢复：会话恢复后激活项可见

**导入导出（IO）**
- 正常：上传导入→进入写作页→导出
- 边界：空文件/无章节标记/编码/大小上限
- 持久化：导入后章节 order_index 连续，导出顺序正确

**结构化资产（OUT/TL/FS/CHAR）**
- 正常：CRUD、排序、关联章节跳转
- 边界：删除章节后引用置空仍可用
- 持久化：刷新后数据仍在

**偏好/专注（PREF）**
- 正常：进入/退出专注、偏好持久化、今日新增口径

## 10. 与 V2 AI 的衔接约束（写死）

- V1.1 只负责沉淀结构化资产，不设计 AI 调用、不设计 Prompt、不设计模型配置
- V1.1 的数据结构要稳定：V2 只能读取这些资产，不得反向破坏用户手写数据
- V2 若需 AI 生成内容，必须使用“候选结果/草稿”机制，禁止直接覆盖 V1.1 手写资产
