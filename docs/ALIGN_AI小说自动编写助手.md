## 一、项目背景

用户已完成142章、30万字传统洪荒背景小说，发布于番茄小说平台。因数据表现不佳，且经腾讯朱雀大模型检测判定80%为AI生成内容，现已断更。

**核心痛点**:
1. 纯AI生成内容易被检测，平台推荐受限
2. 手动创作效率低，难以持续产出
3. 长篇连载（目标800万字）需要极强的剧情连贯性管理

**项目目标**: 开发AI小说助手，基于已有30万字原文和大纲，自动分析并续写，最终完成800万字目标，同时规避AI检测。

## 二、需求边界

### 2.1 一期核心功能（MVP）

| 功能模块 | 需求描述 | 优先级 |
|---------|---------|--------|
| **内容导入** | 上传文件或指定路径，智能解析小说原文和大纲 | P0 |
| **智能解析** | 自动识别章节结构、人物设定、剧情节点，无需预定义格式 | P0 |
| **文风分析** | 全面分析原文：句式结构、用词偏好、修辞手法、对话风格 | P0 |
| **剧情分析** | 分析人物关系、时间线、伏笔、世界观 | P0 |
| **交互式续写** | 分析后构思剧情走向，用户指定章节数或让AI完成剧情 | P0 |
| **连贯性检查** | 严格校验人物状态、剧情时间线、伏笔一致性 | P0 |
| **AI检测规避** | 文风模仿+去AI化处理，降低检测率 | P0 |
| **主备模型** | DeepSeek为主，Kimi为备用，失败时自动切换 | P1 |

### 2.2 二期扩展功能

| 功能模块 | 需求描述 | 优先级 |
|---------|---------|--------|
| **多作品支持** | 支持创建新小说项目，适配不同题材/人设/文风 | P2 |
| **模板系统** | 提供玄幻/仙侠/都市等题材模板 | P2 |
| **人设管理** | 独立的人物设定管理模块 | P2 |
| **世界观管理** | 功法、势力、地图等世界观元素管理 | P2 |

### 2.3 明确不做（Out of Scope）

1. 不直接对接番茄小说API（仅生成内容，人工上传）
2. 不做实时协作/多人编辑
3. 不做图片/封面生成
4. 不做自动发布功能
5. 不要求用户按预定义格式整理文件

## 三、验收标准

### 3.1 功能验收

| 验收项 | 验收标准 | 验证方式 |
|-------|---------|---------|
| 内容导入 | 支持上传文件或指定路径，自动读取内容 | 单元测试 |
| 智能解析 | 自动识别章节结构、人物设定、剧情节点 | 单元测试+人工抽查 |
| 文风分析 | 生成文风特征报告（词汇表、句式模板、修辞统计） | 人工审核 |
| 剧情分析 | 生成人物关系图、时间线、伏笔列表 | 人工审核 |
| 续写质量 | 生成章节与原文风格一致，无明显AI痕迹 | 人工审核+朱雀检测 |
| 连贯性检查 | 人物状态、时间线无矛盾，伏笔正确回收 | 单元测试+人工抽查 |
| 模型切换 | 主模型失败时3秒内切换备用模型 | 集成测试 |

### 3.2 性能验收

| 指标 | 目标值 |
|-----|-------|
| 单章生成时间 | ≤ 60秒 |
| 文风分析（30万字） | ≤ 5分钟 |
| 剧情分析（30万字） | ≤ 10分钟 |
| 连贯性检查（单章） | ≤ 3秒 |
| 系统响应时间 | ≤ 500ms |

### 3.3 质量验收

| 指标 | 目标值 |
|-----|-------|
| 代码测试覆盖率 | ≥ 80% |
| 单元测试通过率 | 100% |
| AI检测率（腾讯朱雀） | ≤ 30% |

## 四、疑问清单

### 4.1 已澄清

| 问题 | 答案 |
|-----|-----|
| 源文件格式 | TXT文本 |
| 章节长度 | 约2100字/章 |
| 文风处理 | 全面分析+模仿 |
| 连贯性检查 | 严格检查 |
| 主模型 | DeepSeek（性价比高，适合大量续写） |
| 备用模型 | Kimi（长上下文，处理复杂分析） |
| 内容导入方式 | 上传文件或指定路径，智能解析 |
| 文件格式要求 | 无预定义格式，工具智能识别 |
| 交互方式 | 分析后交互式续写：构思剧情→指定章节数/完成剧情→生成 |
| 输出格式 | Markdown |
| 现有资料 | 140章30万字小说原文 + 大纲（含人物设定、背景、故事） |
| 界面形式 | 初期Web应用，后期支持桌面 |

### 4.2 待确认

| 编号 | 问题 | 影响 | 建议方案 |
|-----|-----|-----|---------|
| Q1 | 是否需要保存分析结果供后续使用 | 数据持久化 | 建议保存 |

### 4.3 风险预警

| 风险 | 等级 | 缓解措施 |
|-----|-----|---------|
| 30万字文风分析计算量大 | 中 | 采用增量分析，分段处理 |
| AI检测算法更新 | 高 | 持续跟踪检测机制，动态调整策略 |
| 长篇连载连贯性难以保证 | 高 | 建立人物状态库+剧情时间线索引 |
| DeepSeek API稳定性 | 中 | 主备模式+本地缓存 |
| 智能解析准确率 | 中 | 结合规则+AI双重解析 |

## 五、核心工作流

```
┌─────────────────────────────────────────────────────────────┐
│                      用户工作流                              │
├─────────────────────────────────────────────────────────────┤
│  1. 导入内容                                                 │
│     ├── 上传文件（TXT）                                      │
│     └── 或指定文件路径                                       │
│                                                              │
│  2. 智能分析                                                 │
│     ├── 文风分析 → 生成文风特征报告                          │
│     ├── 剧情分析 → 生成人物关系、时间线、伏笔列表            │
│     └── 世界观分析 → 提取设定要素                            │
│                                                              │
│  3. 交互式续写                                               │
│     ├── 用户：提出续写需求（如"写主角突破境界"）             │
│     ├── AI：构思剧情走向，展示给用户确认                     │
│     ├── 用户：指定章节数 或 让AI写完该剧情                   │
│     └── AI：生成章节内容                                     │
│                                                              │
│  4. 审核输出                                                 │
│     ├── 连贯性检查                                           │
│     ├── AI检测率评估                                         │
│     └── 导出Markdown文件                                     │
└─────────────────────────────────────────────────────────────┘
```

### 7.1 术语表

| 术语 | 定义 |
|-----|-----|
| 文风分析 | 对原文语言特征的统计分析，包括词汇、句式、修辞等 |
