# InkTrace 需求规格说明书（v1）

更新时间：2026-04-24  
适用阶段：当前版本

---

## 1. 文档目标

本文件用于指导 InkTrace 当前阶段的产品、设计、开发与测试落地。  
交付目标：稳定可用的长篇写作工具，支持持续编辑、可靠保存、异常可恢复。

---

## 2. 产品定位（当前阶段）

### 2.1 定位

一个稳定支持长篇小说写作的纯文本工具。

### 2.2 范围边界

当前版本必须保证：
- 作品管理可用
- 章节管理可用
- 正文编辑可用
- 自动保存可用
- 关闭恢复可用
- 导入与导出可用

当前版本固定边界：
- 信息架构固定为 `书架 -> 写作页`
- 不新增页面
- 不新增模块
- 编辑器仅支持纯文本

### 2.3 Rule 编号规范（唯一化）

- `R-UI-XX`：界面与交互呈现规则
- `R-CH-XX`：章节管理规则
- `R-EDIT-XX`：编辑器与输入规则
- `R-SAVE-XX`：保存、同步、容错规则
- `R-DATA-XX`：数据、导入、导出规则

---

## 3. 整体结构

信息架构：  
`书架（作品列表） -> 写作页（编辑章节）`

术语统一：
- 使用：`作品`
- 禁用：`项目`

---

## 4. 页面方案

## 4.1 入口页：书架

### 功能清单
- 新建作品
- 导入作品（TXT）
- 查看作品列表
- 打开作品
- 导出作品（.txt）

### 关键交互
- 点击作品卡片进入写作页
- 支持按最近更新时间排序
- 空状态显示“新建作品/导入作品”双入口
- 作品设置或作品卡片提供“导出为 .txt”入口

### 规则落地

Rule（R-CH-04）：新建作品时系统静默创建空章节“第1章”。  
Behavior：创建作品事务内同时写入章节记录（`order_index=1,title='',content=''`）。  
Edge Case：章节创建失败时，作品创建事务整体回滚。

Rule（R-DATA-03）：TXT 导入绝不中断。  
Behavior：未识别任何章节标记时，整本文本按单章节入库。  
Edge Case：空文件导入时创建单章节空内容并返回成功。

Rule（R-DATA-04）：用户可随时导出 `.txt`。  
Behavior：按 `order_index` 顺序拼接章节导出。  
Edge Case：无章节时导出空文本文件并返回成功。

### 验收标准
- 能创建并打开作品
- 导入后作品出现在列表
- 导出入口可用且导出成功
- 列表刷新后不丢失数据

## 4.2 写作页（核心页面）

### 布局
- 左：章节列表
- 中：正文编辑
- 右：默认隐藏

### 关键原则
- 写作为中心
- 状态反馈最小化
- 不打断创作流程

### 规则落地

Rule（R-UI-01）：章节列表支持 500+ 章流畅渲染。  
Behavior：必须使用虚拟滚动或懒加载。  
Edge Case：拖拽后列表可见区与真实顺序一致。

Rule（R-UI-02）：保存状态提示固定三态。  
Behavior：编辑器角落仅显示“保存中/已保存/保存失败”。  
Edge Case：离线期间三态持续可见且不遮挡编辑区。

## 5. 基础功能需求（必须有）

## 5.1 章节管理

### 功能点
- 新建章节
- 删除章节
- 修改标题
- 调整顺序
- 切换章节

### 规则落地

Rule（R-CH-01）：章节显示编号按 `order_index` 动态计算。  
Behavior：展示序号时仅使用当前排序位次。  
Edge Case：拖拽后编号即时重算，刷新后一致。

Rule（R-CH-02）：新建章节默认插入当前编辑章节之后。  
Behavior：插入位为当前章节 `order_index+1`，后续章节顺延。  
Edge Case：当前章节为末章时，追加到末尾。

Rule（R-CH-03）：删除当前编辑章节后必须自动迁移焦点。  
Behavior：优先跳转下一章，无下一章时跳转上一章。  
Edge Case：删除后章节数为 0 时，编辑器进入不可输入空状态。

Rule（R-CH-05）：拖拽调序必须原子提交。  
Behavior：前端一次请求提交受影响章节的 `id+order_index` 全量列表；后端单事务写入。  
Edge Case：事务失败时全量回滚，前端恢复原顺序。

### 基础规则
- 章节标题允许为空（默认显示“未命名章节”）
- 删除章节需二次确认
- 调序后立即持久化

### 验收标准
- 新建后章节立刻出现并可编辑
- 删除后列表与正文同步更新且焦点正确
- 调整顺序后刷新仍保持一致
- 快速切换章节不发生错位

## 5.2 正文编辑

### 功能点
- 输入文本
- 修改文本
- 编辑章节标题

### 规则落地

Rule（R-EDIT-01）：恢复会话必须恢复光标与滚动位置。  
Behavior：进入章节后恢复 `cursor_position` 与 `scroll_top`。  
Edge Case：恢复位置越界时，光标定位文本末尾，滚动定位最大可滚动值。

Rule（R-SAVE-01）：标题与正文同一保存链路提交。  
Behavior：每次 flush 同时提交 `title` 与 `content`。  
Edge Case：任一字段失败时整次保存失败，不更新远端版本。

Rule（R-EDIT-02）：字数统计口径固定为有效字符数。  
Behavior：不计空格、换行、制表符等不可见字符。  
Edge Case：仅不可见字符时字数为 0。

Rule（R-EDIT-03）：单章内容设置性能软上限。  
Behavior：有效字符数超过 200000 时显示顶部软提示，不阻断输入。  
Edge Case：超限后仍可编辑、保存、切章。

Rule（R-EDIT-04）：编辑器仅支持纯文本。  
Behavior：禁用富文本能力，粘贴统一按纯文本处理。  
Edge Case：富文本粘贴仅保留文本字符。

### 验收标准
- 长文可持续编辑
- 撤销/重做正常
- 光标与滚动位置可恢复
- 标题与正文保存一致
- 字数统计口径稳定

## 5.3 基础信息

### 展示项
- 当前章节信息（标题/序号）
- 字数统计（当前章节）

### 规则落地

Rule（R-CH-01）：序号展示与 `order_index` 实时一致。  
Behavior：切章、调序、删除后即时刷新序号。  
Edge Case：并发调序后刷新以最新持久化顺序为准。

Rule（R-EDIT-02）：字数按统一口径实时刷新。  
Behavior：正文变更后立即重算并展示。  
Edge Case：离线状态下统计逻辑不变。

### 验收标准
- 切章后信息实时同步
- 序号与章节顺序一致
- 字数统计与正文一致

## 5.4 数据安全

### 必须能力
- 自动保存
- 切换不丢内容
- 关闭后可恢复

### 规则落地

Rule（R-SAVE-02）：打开作品默认定位上次最后编辑章节。  
Behavior：读取 `last_open_chapter_id` 并自动打开。  
Edge Case：新作品或记录缺失时打开第一章。

Rule（R-SAVE-03）：切章前内容必须进入可恢复状态。  
Behavior：有进行中保存则等待完成或强制 flush；远端失败时先落本地缓存再允许切章。  
Edge Case：本地缓存写入失败时禁止切章并置为“保存失败”。

Rule（R-SAVE-04）：保存采用乐观锁防覆盖。  
Behavior：`chapters.version` 必须参与保存校验；不一致时拒绝覆盖。  
Edge Case：用户选择“覆盖远端”后，以最新远端版本为基准执行覆盖并递增版本。

Rule（R-SAVE-05）：保存失败采用非阻断反馈与静默重试。  
Behavior：仅显示轻量提示，后台按指数退避重试。  
Edge Case：达到重试上限后保留手动重试入口。

Rule（R-SAVE-06）：断网时写作主链路不中断。  
Behavior：进入离线模式后输入实时写入本地缓存；联网后自动 flush 积压草稿。  
Edge Case：回放中断时保留队列，下一次连通继续。

Rule（R-SAVE-07）：本地缓存容量受控。  
Behavior：本地缓存总上限固定 10MB，超限按 LRU 淘汰。  
Edge Case：单章快照过大时先淘汰其他缓存，仍不足则仅保留该章最新快照。

Rule（R-SAVE-08）：本地缓存与远端同步的绝对优先级。  
Behavior：本地草稿实时性优先于远端状态。网络异常或远端保存失败时，以本地缓存为准并持续允许输入。网络恢复后 flush 成功且 `version` 对齐时，以远端确认状态覆盖并清空本地对应快照。触发 `version` 冲突时，本地缓存必须保留，直到用户明确选择“覆盖远端”或“放弃本地”。  
Edge Case：在收到远端成功响应前，禁止清空本地缓存。

### 验收标准
- 异常关闭后可恢复最近编辑
- 连续切章 50 次不丢字
- 重启后可恢复最后章节、光标、滚动位置
- 并发编辑不发生静默覆盖
- 本地缓存不会无限增长

## 6. 数据模型（最小落地）

## 6.1 作品
- id
- title
- author（可空）
- created_at
- updated_at

## 6.2 章节
- id
- work_id
- chapter_no（保留字段，不参与排序）
- title
- content
- word_count
- order_index（排序唯一依据）
- version（乐观锁版本号，默认 1）
- created_at
- updated_at

## 6.3 编辑会话
- work_id
- last_open_chapter_id
- cursor_position
- scroll_top
- last_opened_at

## 6.4 本地缓存
- work_id
- chapter_id
- title_draft
- content_draft
- cursor_position
- scroll_top
- updated_at
- lru_tick

## 7. 状态与异常处理

## 7.1 状态最小化
- 不展示复杂状态面板
- 仅显示“保存中/已保存/保存失败”
- 离线时顶部固定显示“离线模式”

## 7.2 异常策略
- 保存失败不清空编辑区
- 自动重试采用指数退避
- 提供手动重试入口
- 导入解析失败执行单章兜底
- 远端冲突时本地快照保留，直到用户明确决策

---

## 8. 非功能要求

- 性能：10 万字章节流畅输入；500+ 章节列表流畅渲染
- 稳定性：切章、排序、删除不出现白屏
- 可靠性：保存链路幂等；重复请求不污染数据
- 一致性：标题与正文同次 flush；版本冲突可检测
- 可维护性：页面逻辑与存储逻辑分离

## 9. 交付计划

### P0（先可用）
- 书架与写作页基础框架
- 章节 CRUD、切换、调序
- 纯文本编辑、字数统计
- 自动保存、会话恢复、离线缓存

### P1（补稳定）
- 乐观锁冲突处理
- 保存失败重试与状态灯完善
- 本地缓存容量控制与 LRU 淘汰
- 500+ 章节虚拟列表优化

### P2（收口）
- TXT 导入兜底与顺序校正
- `.txt` 导出闭环
- 长章节软上限提示稳定性优化

---

## 10. 验收清单（DoD）

- 可从书架创建并打开作品，创建后自动存在第1章
- 可完成章节新建/删除/改名/调序/切换
- 章节编号始终与 `order_index` 一致
- 新建章节默认插入当前章节后方
- 删除当前章节后焦点按“下优先、上兜底”迁移
- 正文持续可编辑，字数统计口径正确
- 标题与正文每次 flush 同步提交
- 切章前内容进入可恢复状态
- 自动保存生效，切换/关闭/重启后可恢复章节、光标、滚动位置
- 离线模式可持续写作，联网后自动回放
- 并发保存冲突时拒绝静默覆盖
- TXT 导入在任意输入下均可完成
- 作品可随时导出 `.txt`
- 编辑器仅支持纯文本
