# Stage 5 Tasks

### T5-01：正文冲突处理回归

目标：

* 验证正文 409 冲突处理完整链路，确保本地草稿不丢失。

涉及模块：

* Frontend / Backend / Store

实现内容：

* 模拟章节保存返回 `version_conflict`。
* 验证本地草稿保留。
* 验证冲突弹窗展示本地版本与服务端版本。
* 验证覆盖服务端路径。
* 验证放弃本地路径。
* 验证取消后保持冲突状态。

关键规则：

* 冲突未处理前不得清理本地缓存。
* 409 响应必须包含 `server_version`。
* 覆盖服务端使用本地内容重新提交。

完成标准（DoD）：

* 三条冲突决策路径均通过。
* 取消后本地草稿仍存在。
* 覆盖成功后本地草稿清理。

### T5-02：结构化资产冲突处理回归

目标：

* 验证结构化资产 409 冲突处理完整链路。

涉及模块：

* Frontend / Backend / Store

实现内容：

* 模拟结构化资产返回 `asset_version_conflict`。
* 验证 asset_draft 保留。
* 验证 AssetConflictModal 打开。
* 验证覆盖保存路径。
* 验证放弃本地并刷新路径。
* 验证取消路径。

关键规则：

* 结构化资产冲突未决策前不得从远端响应覆盖本地暂存。
* 结构化资产保存成功后必须更新本地版本并清理暂存。
* 冲突弹窗必须提供对比入口。

完成标准（DoD）：

* 结构化资产三条冲突决策路径均通过。
* 取消后 asset_draft 仍存在。
* 放弃本地后远端内容重新加载。

### T5-03：离线写作回归

目标：

* 验证离线状态下正文写作不中断，并在网络恢复后自动同步。

涉及模块：

* Frontend / Store

实现内容：

* 模拟浏览器 offline。
* 输入正文并验证写入本地草稿。
* 验证顶部或状态条显示离线模式。
* 模拟 online。
* 验证草稿按 timestamp 串行回放。
* 验证回放成功后清理草稿。

关键规则：

* 断网时写作主链路不能断。
* 正文离线草稿自动回放。
* 回放成功前不得删除对应本地草稿。

完成标准（DoD）：

* 离线输入不丢字。
* 网络恢复后触发自动同步。
* 同步成功后草稿清理。

### T5-04：结构化资产离线回归

* 网络恢复后不自动提交资产。
* 手动保存成功后暂存清理。

### T5-05：缓存容量与 LRU 回归

目标：

* 验证本地缓存容量上限与 LRU 淘汰策略。

涉及模块：

* Frontend / Store

实现内容：

* 构造超过 10MB 软上限的缓存数据。
* 验证 LRU 淘汰触发。
* 验证结构化资产暂存优先淘汰。
* 验证当前正文草稿不被淘汰。
* 验证冲突状态草稿不被淘汰。
* 验证淘汰提示展示。

关键规则：

* 总容量软上限 10MB。
* 正文草稿优先保留。
* 冲突状态草稿禁止淘汰。

完成标准（DoD）：

* 超限后缓存容量回落。
* 当前正文草稿仍存在。
* 淘汰提示可见。

### T5-06：删除章节引用清理端到端回归

目标：

* 验证删除章节后正文、细纲、时间线、伏笔、大纲节点引用处理一致。

涉及模块：

* Frontend / Backend / DB

实现内容：

* 创建包含多个章节的作品。
* 创建章节细纲、时间线事件、伏笔、全书大纲章节引用。
* 删除被引用章节。
* 验证章节细纲删除。
* 验证时间线、伏笔、全书大纲节点引用置空。
* 验证章节列表 `order_index` 连续。

关键规则：

* 删除章节必须单事务完成。
* 全书大纲节点仅置空引用，不删除节点。
* 删除章节后编辑器不得白屏。

完成标准（DoD）：

* 所有引用清理结果正确。
* 大纲节点仍存在。
* 删除后可继续编辑相邻章节。
