# 执行上下文架构

## 1. 目标

执行上下文架构用于定义：每次分析、规划、续写、改写、校验任务，到底带什么上下文执行。

它和记忆架构不同：

- 记忆架构回答“长期存什么”
- 执行上下文架构回答“这次调用临时拿什么”

如果没有这层设计，系统会出现两个典型问题：

- 数据很多，但不知道本次调用到底该取哪一部分
- token 不够时随意截断，导致分析和续写质量不稳定

## 2. 设计原则

1. 执行上下文必须按任务类型区分
2. 上下文必须从“事实层”优先装配，而不是先拿大量原文
3. 原文只在必要时进入上下文
4. token 预算必须显式分配
5. 丢弃策略必须固定，不能临时拍脑袋

## 3. 上下文层级

单次任务可使用的上下文按优先级分为五层：

1. 锁定约束层
2. 当前任务目标层
3. 结构事实层
4. 近期内容层
5. 原文补充层

### 3.1 锁定约束层

必须优先装入：

- 用户锁定的人物设定
- 用户锁定的世界规则
- 不可改动章节约束
- 当前风格与节奏硬要求

### 3.2 当前任务目标层

包括：

- 当前任务类型
- 当前任务目标
- 当前目标 PlotArc
- 当前章节计划
- 当前写作/改写要求

### 3.3 结构事实层

包括：

- 当前生效 Story Model
- 当前生效 PlotArc
- 当前人物状态
- 当前世界规则摘要
- 当前风险点

### 3.4 近期内容层

包括：

- 最近章节摘要
- 最近章节尾部
- 最近活跃弧推进
- 最近冲突和悬念

### 3.5 原文补充层

包括：

- 当前章节原文
- 相邻章节原文片段
- 必要的大纲原文片段

原文补充层只能在前四层不足以支撑任务时使用。

## 4. 各任务上下文契约

### 4.1 分析任务上下文

分析任务优先使用：

1. 原始导入内容
2. 章节级事实
3. 大纲
4. 已有 Story Model 作为参考，而不是唯一事实

分析任务上下文不应默认塞大量章节原文预览。

推荐输入构成：

- 当前导入批次信息
- 章节列表元数据
- 章节摘要或轻量章节工件
- 大纲
- 已知人物与世界规则

### 4.2 规划任务上下文

规划任务优先使用：

- 当前目标 PlotArc
- 当前章节位置信息
- 当前章节前后关键因果
- 当前未完成目标
- 风格和节奏要求

规划任务不应直接吃整本小说长文本。

### 4.3 续写任务上下文

续写任务优先使用：

- 已确认的当前章节计划
- 当前目标弧
- 最近 1 到 3 章摘要
- 最近一章尾部原文
- 当前人物状态
- 当前世界约束
- 风格约束

续写任务只有在必要时才增加相邻章节片段原文。

### 4.4 改写任务上下文

改写任务必须聚焦在被改写对象。

推荐上下文：

- 被改写章节原文
- 改写目标
- 不可改变的剧情事实
- 不可改变的人物状态
- 风格或节奏要求

改写任务通常不需要整本结构，只需要少量结构约束即可。

### 4.5 校验任务上下文

校验任务需要：

- 当前待校验文本
- 当前章节计划
- 当前 PlotArc
- 当前人物与世界规则
- 最近章节摘要
- 当前 Story Model 风险点

校验任务的目标不是重新写，而是判断是否违背约束。

## 5. 上下文优先级规则

当 token 不足时，优先保留顺序如下：

1. 锁定约束
2. 当前任务目标
3. 当前生效 Story Model 中的核心事实
4. 当前目标 PlotArc
5. 最近章节摘要
6. 最近一章尾部原文
7. 更远章节摘要
8. 原文片段补充

最先被丢弃的应该是：

- 远距离原文
- 可重建展示摘要
- 重复解释性文字

## 6. token 预算策略

### 6.1 总原则

- 先为约束留空间
- 再为目标留空间
- 最后才给长原文

### 6.2 推荐预算分配

以单次任务总预算为 100% 时，建议：

- 约束层：15%
- 任务目标层：15%
- 结构事实层：30%
- 近期内容层：25%
- 原文补充层：15%

具体比例可按任务类型调整。

### 6.3 特殊任务

- 分析任务：结构事实和章节工件预算更高
- 续写任务：近期内容与章节计划预算更高
- 改写任务：原文补充层预算更高
- 校验任务：约束层和结构事实层预算更高

## 7. 长文本处理策略

### 7.1 原则

- 优先章节级拆分
- 其次 chunk 级拆分
- 禁止默认直接整本拼接

### 7.2 适用场景

- 导入分析
- 全书结构重建
- 长篇续写接手

### 7.3 推荐流程

1. 先逐章抽轻量工件
2. 再汇总成全书结构输入
3. 仅在必要时回看原文片段

## 8. 上下文装配器

后续实现中建议明确独立组件，例如：

- `AnalysisContextBuilder`
- `PlanningContextBuilder`
- `WritingContextBuilder`
- `RewriteContextBuilder`
- `ValidationContextBuilder`

它们不应混在单个 service 的 prompt 拼装函数里。

## 9. 上下文可信度要求

如果本次任务的上下文缺少关键元素，则任务不应继续执行。

例如：

- 没有生效 Story Model，不应进入正式续写链
- 没有章节计划，不应直接生成最终正文
- 没有锁定约束加载结果，不应执行改写

## 10. 当前重构要求

重构时必须满足：

1. 不再默认把整本正文或大量 preview 塞进全书分析
2. 全书分析优先基于章节工件
3. 续写上下文优先基于计划、弧、最近章节与约束
4. 改写上下文优先聚焦目标章节，而不是全书
