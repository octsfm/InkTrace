# Workspace 页面级规格说明

## 1. 文档目标

本文件用于把 InkTrace UI 重构中的六个核心页面，进一步收口为页面级规格说明。

这里不再讨论抽象架构，而是明确回答：

- 每个页面到底要解决什么问题
- 每个页面应该放什么
- 不应该放什么
- 页面之间如何协作

这六个页面不是六个独立系统，而是：

**同一个 Workspace 的六个视角。**

其中：

- `Dashboard` 是入口
- `Writing` 是核心
- `Overview` 是理解
- `Structure` 是规划
- `Chapters` 是管理
- `Tasks` 是运行

## 2. 页面总关系

整个系统的主结构如下：

```text
Dashboard
  └── Workspace
        ├── Writing
        ├── Overview
        ├── Structure
        ├── Chapters
        ├── Tasks
        └── Settings
```

统一规则：

1. 所有小说级工作都在 Workspace 内完成。
2. Writing 是默认主视图。
3. 页面切换是“视角切换”，不是“系统切换”。
4. 中央区域始终只放当前最重要的工作对象。

---

## 3. Dashboard

### 3.1 页面目标

Dashboard 的目标只有三个：

1. 快速找到要继续写的小说
2. 一键进入工作区
3. 轻量管理小说

### 3.2 页面结构

#### 顶部

轻量 Header，仅包含：

- Logo / 产品名
- 全局搜索 / Command Palette（`Cmd+K` / `Ctrl+K`）
- 新建小说按钮
- 导入小说按钮
- 用户菜单

规则：

- 不使用大面积背景色
- 不使用厚重导航栏
- 不堆过多系统级入口

#### 主区域

主区域采用卡片流（Grid）。

每张卡片结构：

- 小说名（主标题）
- 标签（题材）
- 状态（创作中 / 已完成 / 停滞）
- 当前进度（如：第 142 章 / 32.7 万字）
- 最近更新时间
- 主按钮：继续写作

#### 可选辅助区

可在顶部加入：

- 最近编辑
- 有未完成任务的小说

但不能抢主视觉。

### 3.3 交互规则

- 点击卡片 -> 直接进入 Workspace（默认 Writing）
- 点击“继续写作” -> 打开上次编辑章节
- `Cmd+K` / `Ctrl+K` -> 可直接搜索章节 / 小说并跳转

### 3.4 空状态

无小说时显示：

- 创建小说
- 导入小说
- 从大纲开始

### 3.5 不应承担的职责

Dashboard 不负责：

- 正文编辑
- 深度结构浏览
- 复杂任务操作

它只负责进入小说，不负责完成小说。

## 4. Writing

### 4.1 页面目标

Writing 是整个系统的核心页面，目标是：

1. 承载写作
2. 承载 AI 协作
3. 成为默认入口

### 4.2 页面布局

采用固定结构：

`[主导航栏] [章节树] [编辑器] [Copilot]`

### 4.3 中央编辑器

中央编辑器是整个页面的绝对核心。

编辑器内容：

- 当前章节正文
- 支持富文本
- 支持 AI 插入

编辑器状态至少包括：

- 正式文本
- 候选文本（AI 生成）
- 正在生成
- issue 标记

### 4.4 行内操作

#### 选区菜单

选中一段后，应支持：

- 润色
- 扩写
- 缩写
- 改写
- 去 AI 味

#### Slash 命令

输入 `/` 后，应支持：

- `/续写`
- `/生成下一段`
- `/补环境`
- `/增强冲突`

### 4.5 候选块

AI 生成内容以候选形式出现，不直接覆盖正式正文。

候选块支持：

- 接受
- 局部接受
- 修改后接受
- 拒绝

### 4.6 左侧章节树

功能：

- 显示章节目录
- 当前章节高亮
- 支持搜索
- 支持折叠

### 4.7 右侧 Copilot

Tab 结构：

- 对话
- 上下文
- 灵感

#### 对话

自然语言协作。

#### 上下文

显示：

- 当前章节
- 当前 PlotArc
- 当前人物
- 当前阶段

#### 灵感

显示：

- 下一章建议
- 冲突建议
- 改写方向

### 4.8 空状态

- 未选章节 -> 提示选择或创建
- 空章节 -> 提供写作建议

### 4.9 状态显示

顶部或底部显示：

- 自动保存状态
- 当前字数
- 任务状态

### 4.10 页面规则

- Writing 是默认入口
- 中央只能是编辑器，不允许被仪表盘和说明卡片占满
- 所有 AI 行为必须绑定当前章节

---

## 5. Overview

### 5.1 页面目标

Overview 的目标是：

1. 快速理解小说当前状态
2. 提供下一步建议

### 5.2 页面结构

建议分为四块：

#### 当前状态

- 小说名
- 状态
- 字数
- 章节数

#### 最近进度

- 最近编辑章节
- 最近生成内容
- 最近更新时间

#### 当前建议

- 继续某章节
- 查看结构问题
- 处理未完成任务

#### 近期任务

- 正在运行
- 最近失败
- 最近完成

### 5.3 设计要求

- 允许轻卡片
- 不做仪表盘墙
- 不抢写作优先级

### 5.4 不应承担的职责

Overview 不应变成：

- 旧详情页的复刻
- 功能入口堆叠页
- 大量重型分析卡片的容器

---

## 6. Structure

### 6.1 页面目标

Structure 的目标是：

1. 理解小说结构
2. 支撑规划

### 6.2 左侧导航

左二栏包括：

- Story Model
- PlotArc
- 人物
- 世界观
- 风险点

### 6.3 中央内容

#### Story Model

展示：

- 主线
- 当前阶段
- 风格
- 风险

#### PlotArc

展示：

- 活跃弧
- 阶段弧
- 关联章节

#### 人物

展示：

- 人物信息
- 最近出现章节

#### 世界观

展示：

- 规则
- 设定

### 6.4 页面规则

- Structure 不抢写作主流程
- 它服务分析与规划，而不是成为默认入口
- 第一阶段优先做清晰信息结构，不先追求复杂动态图谱

---

## 7. Chapters

### 7.1 页面目标

Chapters 的目标是：

1. 管理章节结构
2. 支持批量操作

### 7.2 展示模式

支持两种模式：

#### 列表模式

显示：

- 章节顺序
- 更新时间
- 状态

#### 看板模式

按状态分类：

- 构思中
- 草稿
- 已校验
- 已完成

### 7.3 功能

- 新建章节
- 删除
- 排序
- 移动
- 批量操作

### 7.4 与 Writing 联动

- 点击章节 -> 打开 Writing
- 修改 -> 实时同步

### 7.5 页面规则

- Chapters 负责章节管理，不负责正文创作
- 章节树和章节管理列表要共享同一份章节对象状态

---

## 8. Tasks

### 8.1 页面目标

Tasks 的目标是：

1. 管理所有任务
2. 提供恢复能力

### 8.2 页面结构

#### 当前任务

- 正在执行
- 显示进度

#### 最近失败

- 显示错误
- 提供重试

#### 历史任务

- 已完成记录

### 8.3 单任务信息

每个任务至少显示：

- 名称
- 类型
- 状态
- 进度
- 时间
- 详情

### 8.4 审查任务

任务页要承载：

- issue 列表
- 锚点定位
- 修复入口

### 8.5 失败处理

失败任务必须提供：

- 错误原因
- 重试按钮
- 恢复路径

### 8.6 页面规则

- Tasks 不是日志页
- Tasks 是系统运行状态与恢复入口页
- 所有长任务必须能在这里被看见和被恢复

---

## 9. 统一规则

以下规则适用于所有页面：

### 9.1 不跳页面

所有操作尽量在 Workspace 内完成。

### 9.2 中央唯一原则

中央只放当前工作内容。

### 9.3 AI 绑定上下文

所有 AI 行为必须绑定当前章节。

### 9.4 写作优先

优先级为：

`写作 > 改写 > 审查 > 浏览`

### 9.5 数据一致性

- 正式文本与候选文本分离
- 状态跨视图同步

---

## 10. 最终总结

这六个页面不是六个系统，而是：

**同一个工作区的六个视角。**

其中：

- `Writing` 是核心
- `Overview` 是理解
- `Structure` 是规划
- `Chapters` 是管理
- `Tasks` 是运行
- `Dashboard` 是入口

整个 InkTrace UI 重构，最终要收口成一句话：

**Dashboard 负责进入小说，Workspace 负责完成小说。**
