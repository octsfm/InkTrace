## 一、项目概述

### 1.1 项目背景

用户已完成142章、30万字传统洪荒背景小说，发布于番茄小说平台。因数据表现不佳，且经腾讯朱雀大模型检测判定80%为AI生成内容，现已断更。

**核心痛点**:
1. 纯AI生成内容易被检测，平台推荐受限
2. 手动创作效率低，难以持续产出
3. 长篇连载（目标800万字）需要极强的剧情连贯性管理

**项目目标**: 开发AI小说助手，基于已有30万字原文和大纲，自动分析并续写，最终完成800万字目标，同时规避AI检测。

### 1.2 核心价值

| 价值点 | 描述 |
|-------|-----|
| 文风模仿 | 分析原文风格，生成内容与原文一致，规避AI检测 |
| 连贯性保证 | 严格检查人物状态、时间线、伏笔一致性 |
| 交互式续写 | 用户参与剧情规划，AI负责内容生成 |
| 主备模型 | DeepSeek主 + Kimi备，保证服务稳定性 |

## 二、需求确认

### 2.1 一期核心功能（MVP）

| 功能模块 | 需求描述 | 优先级 |
|---------|---------|--------|
| **内容导入** | 上传文件或指定路径，智能解析小说原文和大纲 | P0 |
| **智能解析** | 自动识别章节结构、人物设定、剧情节点，无需预定义格式 | P0 |
| **文风分析** | 全面分析原文：句式结构、用词偏好、修辞手法、对话风格 | P0 |
| **剧情分析** | 分析人物关系、时间线、伏笔、世界观 | P0 |
| **交互式续写** | 分析后构思剧情走向，用户指定章节数或让AI完成剧情 | P0 |
| **连贯性检查** | 严格校验人物状态、剧情时间线、伏笔一致性 | P0 |
| **AI检测规避** | 文风模仿+去AI化处理，降低检测率 | P0 |
| **主备模型** | DeepSeek为主，Kimi为备用，失败时自动切换 | P1 |

### 2.2 验收标准

| 验收项 | 验收标准 |
|-------|---------|
| 内容导入 | 支持上传文件或指定路径，自动读取内容 |
| 智能解析 | 自动识别章节结构、人物设定、剧情节点 |
| 文风分析 | 生成文风特征报告（词汇表、句式模板、修辞统计） |
| 剧情分析 | 生成人物关系图、时间线、伏笔列表 |
| 续写质量 | 生成章节与原文风格一致，无明显AI痕迹 |
| 连贯性检查 | 人物状态、时间线无矛盾，伏笔正确回收 |
| 模型切换 | 主模型失败时3秒内切换备用模型 |
| 单章生成时间 | ≤ 60秒 |
| 文风分析（30万字） | ≤ 5分钟 |
| 代码测试覆盖率 | ≥ 80% |
| AI检测率（腾讯朱雀） | ≤ 30% |

### 2.3 关键决策

| 决策项 | 选择 | 理由 |
|-------|-----|-----|
| 源文件格式 | TXT文本 | 用户现有文件格式 |
| 章节长度 | 2100字/章 | 用户指定 |
| 主模型 | DeepSeek | 性价比高，中文创作能力强 |
| 备用模型 | Kimi | 长上下文，处理复杂分析 |
| 内容导入 | 上传文件或指定路径，智能解析 | 无需预定义格式 |
| 交互方式 | 分析后交互式续写 | 用户参与剧情规划 |
| 输出格式 | Markdown | 用户指定 |
| 界面形式 | 初期Web应用，后期支持桌面 | 用户指定 |

## 三、架构确认

### 3.1 分层架构

```
表现层 (FastAPI + Vue3)
    ↓
应用层 (业务逻辑编排)
    ↓
领域层 (聚合/实体/值对象/领域服务/仓储接口)
    ↓
基础设施层 (大模型/文件/数据库/缓存适配)
```

### 3.2 核心领域模型

| 类型 | 模型 |
|-----|-----|
| 聚合根 | Novel, Outline |
| 实体 | Chapter, Character, Plot, Timeline, Foreshadowing |
| 值对象 | StyleProfile, CharacterState, PlotNode |
| 领域服务 | StyleAnalyzer, PlotAnalyzer, ConsistencyChecker, WritingEngine |

### 3.3 技术栈

| 层级 | 技术选型 |
|-----|---------|
| 表现层 | FastAPI + WebSocket + Vue3 |
| 应用层 | Python 3.11+ |
| 领域层 | Python + Pydantic |
| 基础设施层 | httpx + SQLite |
| 大模型 | DeepSeek API + Kimi API |
| 测试 | pytest + pytest-cov |
