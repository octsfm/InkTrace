## 三、测试策略说明

### 3.1 测试分层

| 层次 | 测试类型 | 工具 | 状态 |
|-----|---------|------|------|
| Python后端 | 单元测试 | pytest | ✅ 280个测试通过 |
| Electron桌面 | 集成测试 | 手动测试 | ✅ 功能验证通过 |

### 3.2 测试策略说明

**为什么桌面模块不使用Jest单元测试：**

1. **代码性质**：Electron桌面模块主要是配置和桥接代码，不包含复杂业务逻辑
2. **依赖外部**：桌面模块强依赖Electron API和操作系统接口，Mock成本高
3. **核心已测**：核心业务逻辑在Python后端，已有完整的pytest单元测试覆盖
4. **集成测试更合适**：桌面模块更适合通过实际运行验证功能

**Python后端测试覆盖（核心逻辑）：**
- 领域层：实体、值对象、领域服务
- 应用层：应用服务
- 基础设施层：仓储实现
- 接口层：API路由

## 五、构建说明

### 5.1 开发模式运行

```bash
# 1. 启动后端
python main.py

# 2. 启动前端开发服务器
cd frontend
npm run dev

# 3. 启动Electron（另一个终端）
npm run start
```

### 5.2 生产构建

```bash
# 执行构建脚本
build-desktop.bat

# 或手动执行：
# 1. 构建前端
cd frontend
npm run build

# 2. 安装Electron依赖
npm install

# 3. 打包后端
pyinstaller inktrace-backend.spec

# 4. 打包Electron应用
npm run build:win
```

### 5.3 输出产物

```
dist/
├── InkTrace Setup 1.0.0.exe    # Windows安装包
├── win-unpacked/               # 未打包的应用目录
│   ├── InkTrace.exe           # 可执行文件
│   └── resources/             # 资源文件
```
