## 总目标
交付一个稳定可用的纯文本写作工具（书架 + 写作页），具备：章节 CRUD、拖拽调序、自动保存（Local-First）、会话恢复（章节/光标/滚动）、离线不中断、导入/导出 TXT、乐观锁冲突处理（P1）。

## 必须遵守的硬约束
- 严格按开发计划 Stage 0 → Stage 1(P0) → Stage 2(P1) → Stage 3(P2) 逐项实现；一次只做一个任务编号，做完必须可运行可验证再进入下一项。
- 自动保存链路必须写死为：**输入 → 立即写本地缓存 → debounce(2-3s) → API 同步 → 成功后清理本地缓存**。
- P0 核心验收必须达成：**连续输入 → 3 秒内自动保存完成 → 刷新后内容不丢失**。
- Store 必须拆分：`useWorkspaceStore`（EditSession）、`useChapterDataStore`（数据）、`useSaveStateStore`（状态机与同步）。禁止合并成一个大 Store。
- 离线回放必须：按 timestamp 排序 + 串行 replay，禁止并发打爆接口。
- V1 阶段右侧栏：默认隐藏，不允许展开复杂面板（UI 文档有硬约束）。
