# InkTrace 界面与交互设计文档 (UI/UE v1.1)

更新时间：2026-04-27
适用阶段：当前版本

---

## 1. 设计目标与原则

基于《InkTrace 基础方案落地文档（v1.1）》，本系统的界面设计必须遵循以下核心原则：

1. **沉浸式创作优先**：最大化纯文本编辑区，弱化所有非核心控制元素。
2. **极简状态反馈**：用最少的视觉干扰（如小圆点、角落文字）传达系统状态，杜绝阻断式弹窗（除非遇到无法挽回的错误或冲突）。
3. **响应式与性能优先**：在大纲/章节侧边栏支撑 500+ 列表时，保持滚动和拖拽的丝滑体验。

---

## 2. 全局样式规范 (Design System)

- **主色调**：
  - 背景色：极简灰白 `#FAFAFA` (白天模式) / 深灰 `#1E1E1E` (夜间模式预留)。
  - 侧边栏/控制区背景：纯白 `#FFFFFF`。
  - 主文本色：深灰黑 `#333333`（提高长时间阅读舒适度）。
  - 次要文本色（状态/统计）：浅灰 `#999999`。
- **状态色**：
  - 成功/已保存：绿色 `#52C41A`。
  - 进行中/保存中：蓝色 `#1890FF` 或 灰色 Loading 动画。
  - 警告/离线/失败：橙红色 `#FA8C16`。
- **排版 (Typography)**：
  - 章节列表字体：`14px`，无衬线或系统默认 UI 字体。
  - 正文编辑字体：`16px` 或 `18px`，行高 `1.8`，推荐使用护眼的等宽字体或类似宋体/黑体的阅读字体。

## 3. 核心页面线框图 (Wireframes)

### 3.1 页面一：书架 (WorksList.vue)

作为进入系统的第一入口，界面需保持整洁，突出“新建”与“导入”核心动作。

```text
+-----------------------------------------------------------------------------+
|  [ InkTrace Logo ]                                            [ 设置 ] [ 退出 ] |
|-----------------------------------------------------------------------------|
|                                                                             |
|  【 我的作品 】 (按最近更新排序)                                               |
|                                                                             |
|  +--------------------+  +--------------------+  +--------------------+     |
|  | [+] 新建作品       |  | [txt] 导入作品     |  | 📖 星辰变            |     |
|  |                    |  |                    |  |                    |     |
|  | (点击创建空白工程)  |  | (解析TXT并分章)     |  | 作者: 我吃西红柿    |     |
|  |                    |  |                    |  | 字数: 280 万       |     |
|  +--------------------+  +--------------------+  | 更新: 10分钟前      |     |
|                                                  |                    |     |
|                                                  | [ 导出TXT ] [ 删除 ]|     |
|                                                  +--------------------+     |
|                                                                             |
|  +--------------------+                                                     |
|  | 📖 凡人修仙传      |                                                     |
|  | 作者: 忘语          |                                                     |
|  | 字数: 15 万         |                                                     |
|  | 更新: 2天前         |                                                     |
|  |                    |                                                     |
|  | [ 导出TXT ] [ 删除 ]|                                                     |
|  +--------------------+                                                     |
+-----------------------------------------------------------------------------+
```

**交互规范 (UE)**：
- **卡片点击**：点击“新建”直接触发接口，成功后直接跳转进入写作页；点击已有作品卡片直接跳转进入写作页（自动定位上次阅读的章节）。
- **导入动作**：点击“导入”，弹出系统文件选择框。选择文件后，显示非阻断的全局进度条，完成后自动刷新列表。
- **删除动作**：点击删除，弹出**二次确认框**（“此操作不可恢复，确认删除？”）。

### 3.2 页面二：写作页 (WritingStudio.vue)

这是系统的灵魂页面。布局分为 `侧边栏 (20%)` + `编辑区 (80%)`。

```text
+-----------------------------------------------------------------------------+
| [← 返回书架]  作品: 星辰变                                  ( 离线模式 ⚠️ ) |
|-------------------------+---------------------------------------------------|
| 【 章节列表 (共 500 章) 】|                                                   |
|                         |  [输入章节标题...]                           (已保存) |
| [+] 新增章节            |                                                   |
| ----------------------- |  ===============================================  |
| ≡ 第1章 陨星降落        |  |                                               |  |
| ----------------------- |  | 陨星降落在炎京城外，秦羽...                     |  |
| ≡ 第2章 潜龙在渊        |  |                                               |  |
| ----------------------- |  | (这里是纯文本输入区域，支持顺滑滚动)              |  |
| ≡ 第3章 星辰变 (当前)   |  |                                               |  |
| ----------------------- |  |                                               |  |
| ≡ 第4章 飞升神界        |  |                                               |  |
| ----------------------- |  |                                               |  |
| ... (虚拟滚动)          |  |                                               |  |
|                         |  ===============================================  |
|                         |                                                   |
|                         |  本章字数: 3,421 字                                |
+-------------------------+---------------------------------------------------+
```

#### 3.2.1 左侧：章节侧边栏 (ChapterSidebar)
**视觉设计**：
- 列表项高度固定（如 `40px`），文字单行截断（超出显示 `...`）。
- 当前激活的章节高亮显示（如背景色加深或左侧带有蓝色标识条）。
- 每个列表项左侧带有一个极简的“拖拽手柄”图标 (`≡` 或 `⋮⋮`)。

**交互规范 (UE)**：
- **新增章节**：点击顶部 `[+]`，在**当前激活章节的下方**插入一个“未命名章节”，并自动切换到该新章节，光标聚焦到右侧标题输入框。
- **删除章节**：鼠标 Hover 到列表项上时，右侧浮现垃圾桶图标。点击后弹出二次确认框。
- **拖拽排序**：按住左侧手柄拖拽。拖拽过程中显示占位虚线框。松手后，列表重新渲染，触发防抖的全量排序接口。

#### 3.2.2 右侧：沉浸式编辑区 (PureTextEditor)
**视觉设计**：
- 顶部保留大号字体（如 `24px`，加粗）作为标题输入框。
- 正文输入区去边框，占满剩余高度（`flex-grow: 1`），内部添加适当的内边距（如 `padding: 20px 40px`）以提供“纸张感”。
- 状态指示器悬浮在右上角或右下角（避免遮挡文字）。
- 字数统计固定在右下角。

**交互规范 (UE)**：
- **纯文本输入**：拦截一切富文本格式，只允许输入纯文本。
- **状态机反馈**：
  - 用户打字时：右上角状态立即变为“保存中...”。
  - 停止打字 2 秒后：触发接口，成功后变为“已保存 ✔”。
  - 断网或接口失败：变为红色的“保存失败/离线暂存 ⚠️”。
- **会话恢复**：从侧边栏点击切换章节时，编辑区的滚动条和光标必须瞬间跳到上次离开的位置。

## 4. 关键交互弹窗与反馈 (Modals & Toasts)

### 4.1 乐观锁冲突弹窗 (Version Conflict Modal)

当发生并发覆盖危险（API 返回 409）时，必须阻断用户继续提交，强制决策。

```text
+-------------------------------------------------+
| ⚠️ 检测到版本冲突                               |
|-------------------------------------------------|
| 您当前编辑的章节，在云端已经被其他设备或标签页  |
| 更新过。                                        |
|                                                 |
| 如果继续保存，将会覆盖云端的最新内容。          |
|                                                 |
|  [ 放弃本地修改，重新加载云端 ]  [ 强制覆盖云端 ] |
+-------------------------------------------------+
```

### 4.2 轻量级提示 (Toast)
用于非阻断式的信息反馈（显示 3 秒后消失）：
- “作品导出成功，开始下载” (绿色)
- “导入成功，共解析 120 章” (绿色)
- “网络已断开，已开启本地缓存模式” (橙色)

## 5. 遗留 UI 清理说明

基于新的极简纯文本设计理念，开发时需**彻底移除**以下 V1 遗留 UI 组件：
- 移除所有与 AI 对话、剧情弧线（Arc）相关的抽屉或浮窗。
- 移除富文本编辑器（TipTap）及其悬浮菜单（BubbleMenu）。
- 移除右侧复杂的“设置”、“知识库”、“人物卡”侧边栏。
