# InkTrace 界面与交互设计文档 (UI/UE v1.1)

更新时间：2026-04-27
适用阶段：当前版本

---

## 1. 设计目标与原则

基于《InkTrace 基础方案落地文档（v1.1）》，本系统的界面设计必须遵循以下核心原则：

1. **沉浸式创作优先**：最大化纯文本编辑区，弱化所有非核心控制元素。
2. **极简状态反馈**：用最少的视觉干扰（如小圆点、角落文字）传达系统状态，杜绝阻断式弹窗（除非遇到无法挽回的错误或冲突）。
3. **响应式与性能优先**：在大纲/章节侧边栏支撑 500+ 列表时，保持滚动和拖拽的丝滑体验。

---

## 2. 全局样式规范 (Design System)

- **主色调**：
  - 背景色：极简灰白 `#FAFAFA` (白天模式) / 深灰 `#1E1E1E` (夜间模式预留)。
  - 侧边栏/控制区背景：纯白 `#FFFFFF`。
  - 主文本色：深灰黑 `#333333`（提高长时间阅读舒适度）。
  - 次要文本色（状态/统计）：浅灰 `#999999`。
- **状态色**：
  - 成功/已保存：绿色 `#52C41A`。
  - 进行中/保存中：蓝色 `#1890FF` 或 灰色 Loading 动画。
  - 警告/离线/失败：橙红色 `#FA8C16`。
- **排版 (Typography)**：
  - 章节列表字体：`14px`，无衬线或系统默认 UI 字体。
  - 正文编辑字体：`16px` 或 `18px`，行高 `1.8`，推荐使用护眼的等宽字体或类似宋体/黑体的阅读字体。

## 3. 核心页面线框图 (Wireframes)

### 3.1 页面一：书架 (WorksList.vue)

作为进入系统的第一入口，界面需保持整洁，突出“新建”与“导入”核心动作。参考阅文作家助手的卡片式排版与顶部快捷操作栏。

```text
+-----------------------------------------------------------------------------+
|  [ InkTrace Logo ]                                            [ 设置 ] [ 退出 ] |
|-----------------------------------------------------------------------------|
|                                                                             |
|  +------------------------+  +------------------------+                     |
|  | 📄 新建                  |  | ⬇️ 导入                  |                     |
|  | 私密作品、阅文作品、分组 |  | 本地导入私密作品          |                     |
|  +------------------------+  +------------------------+                     |
|                                                                             |
|                                                               👁️ 已隐藏  🗑️ 回收站 |
|  +-------------------+  +-------------------+  +-------------------+        |
|  | 📖 星辰变           |  | 📖 凡人修仙传      |  | 📖 修仙从儿时开始    |        |
|  |                   |  |                   |  |                   |        |
|  |                   |  |                   |  |                   |        |
|  |                   |  |                   |  |                   |        |
|  +-------------------+  +-------------------+  +-------------------+        |
|  星辰变         (...)   凡人修仙传       (...)   修仙从儿时开始     (...)   |
|  280.4万字              15.2万字               1.9万字                      |
+-----------------------------------------------------------------------------+
```

**视觉与交互规范 (UI/UE)**：
- **顶部操作栏**：横向排列的大按钮（新建、导入），带有轻微阴影，悬浮时背景色微变。
- **作品卡片排版**：
  - 封面区域为主视觉（默认显示带有羽毛/笔图标的纯色背景，左上角可带状态标签如“私密”）。
  - 卡片下方独立显示作品名称和字数。
  - 名称右侧带有 `(...)` 更多操作按钮。
- **更多操作菜单**：点击 `(...)` 弹出下拉菜单，包含：
  - 导出 TXT
  - 重命名
  - 删除（点击后弹出二次确认框：“此操作不可恢复，确认删除？”）
- **跳转动作**：点击作品卡片封面或名称，直接跳转进入写作页（自动定位上次阅读的章节）。

### 3.2 页面二：写作页 (WritingStudio.vue)

这是系统的灵魂页面。布局参考阅文作家助手，采用 `极简左侧边栏` + `中央沉浸式编辑区` + `极简右侧边栏（预留/扩展）`。

#### 3.2.1 左侧：章节侧边栏 (ChapterSidebar)
**视觉设计**：
- 顶部固定 `[新建章节]` 按钮（大按钮，显著突出）。
- 列表支持过滤/分组（如“草稿”、“已发布”选项卡，V1可仅保留单列表）。
- 列表项高度紧凑，左侧显示标题，**右侧对齐显示字数**（如 `2122`）。
- 当前激活的章节背景色微变（如极浅的蓝色/灰色）。

**交互规范 (UE)**：
- **新增章节**：点击顶部 `[新建章节]` 或底部 `+ 新建章节`，在末尾或当前激活章节的下方插入一个“第X章”，并自动聚焦到中央编辑区。
- **删除章节**：列表顶部提供批量管理按钮（垃圾桶图标），或 Hover 列表项时浮现操作图标。
- **拖拽排序**：长按列表项即可拖拽排序，松手后触发接口。

#### 3.2.2 中央：沉浸式编辑区 (PureTextEditor)
**视觉设计**：
- 顶部为大号字体的标题输入框（如 `第142章 大陆，起！`）。
- 正文区域两侧留白极宽（类似 Word 页面居中），提升阅读体验，单行字数控制在最佳阅读区间。
- 底部状态栏显示字数统计（如 `本章: 2070`）。

**交互规范 (UE)**：
- **纯文本输入**：完全拦截富文本。支持顺滑的滚轮操作。
- **状态机反馈**：顶部 Header 右侧显示同步状态（“已同步”/“保存中...”/“网络异常”）。
- **打字机模式 (可选)**：保持当前输入行始终位于屏幕垂直中央。

#### 3.2.3 右侧：快捷工具栏 (预留)
**视觉设计**：
- 极窄的图标列（如宽度 `50px`）。
- 包含大纲、角色、设定等扩展入口。
- **V1 阶段处理**：UI 上可以画出这些图标占位，但点击时不弹出复杂面板，或者直接在 V1 阶段隐藏整个右侧栏以保证绝对的极简。

## 4. 关键交互弹窗与反馈 (Modals & Toasts)

### 4.1 乐观锁冲突弹窗 (Version Conflict Modal)

当发生并发覆盖危险（API 返回 409）时，必须阻断用户继续提交，强制决策。

```text
+-------------------------------------------------+
| ⚠️ 检测到版本冲突                               |
|-------------------------------------------------|
| 您当前编辑的章节，在云端已经被其他设备或标签页  |
| 更新过。                                        |
|                                                 |
| 如果继续保存，将会覆盖云端的最新内容。          |
|                                                 |
|  [ 放弃本地修改，重新加载云端 ]  [ 强制覆盖云端 ] |
+-------------------------------------------------+
```

### 4.2 轻量级提示 (Toast)
用于非阻断式的信息反馈（显示 3 秒后消失）：
- “作品导出成功，开始下载” (绿色)
- “导入成功，共解析 120 章” (绿色)
- “网络已断开，已开启本地缓存模式” (橙色)

## 5. 遗留 UI 清理说明

基于新的极简纯文本设计理念，开发时需**彻底移除**以下 V1 遗留 UI 组件：
- 移除所有与 AI 对话、剧情弧线（Arc）相关的抽屉或浮窗。
- 移除富文本编辑器（TipTap）及其悬浮菜单（BubbleMenu）。
- 移除右侧复杂的“设置”、“知识库”、“人物卡”侧边栏。
