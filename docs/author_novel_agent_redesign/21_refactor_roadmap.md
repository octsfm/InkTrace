# 重构路线图

## 用途

把重构设计转化为实际实现顺序。

## 阶段划分

### Phase 0

- 冻结旧页面上的继续堆功能行为
- 把当前仓库视作零件库
- 确认蓝图与设计基线

### Phase 1

- 搭建新架构骨架
- 明确状态、版本、存储、上下文契约
- 建立最小智能体闭环

### Phase 2

- 重做导入与接手链
- 重做 Story Model 与 PlotArc 链
- 重做写作、校验、修订闭环

### Phase 3

- 按页面职责拆分前端
- 切到概览 / 结构 / 章节 / 写作 / 任务模式
- 清晰暴露智能体驱动流程

### Phase 4

- 对长篇续写可靠性做完整验证
- 强化质量控制与可观测性
- 清退旧路径

## 迁移规则

- 可复用稳定底层资产
- 不直接继承旧页面结构
- 不再以旧产品假设为基线

## 最终交付

- 新架构地图
- 新服务边界
- 新任务与状态模型
- 新页面结构
- 旧模块废弃清单
