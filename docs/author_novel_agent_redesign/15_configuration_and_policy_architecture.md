# 配置与策略架构

## 1. 目标

配置与策略架构用于定义 InkTrace 在长期运行中哪些行为是可配置的、哪些规则是固定的、哪些策略可以按小说类型或任务类型切换，以及这些配置如何稳定影响智能体、工作流与模型调用。

本架构的目标不是“把所有行为都做成开关”，而是建立一套可控、可追溯、可渐进优化的策略系统，避免关键规则散落在代码里的 `if/else` 与隐式默认值中。

---

## 2. 设计原则

### 2.1 规则分层

系统规则分为四层：

1. 固定规则  
   由产品原则或安全约束决定，不允许普通配置覆盖。

2. 平台默认策略  
   针对所有小说生效的默认策略，可随版本演进。

3. 小说级策略  
   针对某一本小说的长期策略设置。

4. 单次任务策略  
   针对某一次分析、规划、续写、改写、校验任务的临时覆盖。

优先级从高到低为：

`固定规则 > 单次任务策略 > 小说级策略 > 平台默认策略`

但固定规则不可被下层覆盖。

### 2.2 配置不是实现细节泄漏

只有真正需要长期调优、跨小说复用、跨版本比较的参数才允许进入配置层。  
纯实现细节不应暴露为配置。

### 2.3 配置必须可追溯

任何会影响分析、规划、写作、改写、校验结果的策略，都必须能在任务记录和结果版本中追溯到：

- 使用了哪一份策略
- 使用了哪个版本
- 是否有临时覆盖

### 2.4 配置必须有安全兜底

系统必须定义“不可被关闭”的底线规则，例如：

- 分析职责必须走 Kimi
- 正文写作职责必须走 DeepSeek
- 低可信分析不得直接进入写作链
- 被锁定内容不得被自动覆盖

这些规则属于固定规则，不进入普通策略配置。

---

## 3. 配置对象分层

### 3.1 平台默认策略（Platform Policy）

平台默认策略定义系统在无特殊设置时的标准行为，适用于：

- 默认导入模式
- 默认上下文预算
- 默认写作链
- 默认校验阻断阈值
- 默认 PlotArc 激活窗口

平台默认策略是系统稳定运行的主基线。

### 3.2 小说级策略（Novel Policy）

小说级策略定义某一本小说在整个创作周期内的长期策略，例如：

- 默认写作风格
- 默认节奏偏好
- 默认规划模式
- 默认目标弧偏好
- 是否允许激进改写
- 是否启用更严格的一致性阻断

### 3.3 任务级策略（Task Policy）

任务级策略定义某一次任务的临时覆盖，例如：

- 本次改写使用“保守模式”
- 本次规划强制使用 `deep_planning`
- 本次分析限制为最近 20 章
- 本次续写优先推进指定 PlotArc

任务级策略只影响当前任务，不应无意写回小说长期配置。

### 3.4 Prompt 与上下文策略

Prompt 版本和上下文装配规则也属于策略系统的一部分，必须纳入版本追踪，不允许作为隐藏实现逻辑存在。

---

## 4. 配置项分类

### 4.1 分析策略配置

分析类任务的配置至少包括：

- 导入时默认使用 `full_reanalyze / rebuild_global / refresh_view` 哪种模式
- 全书分析的汇总层级
- 每次分析允许采样的章节数量上限
- 单章分析是否启用深度分析
- 低可信分析的阻断阈值
- 是否启用自动补全缺失结构项

### 4.2 PlotArc 策略配置

PlotArc 策略至少包括：

- 活跃弧最小数量
- 活跃弧最大数量
- 默认弧类型优先级
- 阶段跳转约束
- 是否允许自动恢复休眠弧
- 最近弧窗口长度
- 全书弧 / 阶段弧 / 最近弧的默认生成策略

### 4.3 规划策略配置

规划类策略至少包括：

- 默认规划模式：`light_planning / deep_planning`
- 哪些条件下自动升级到深规划
- 单章目标数量上限
- 是否允许同章推进多条弧
- 章节功能多样性约束
- 规划失败后的重试或降级规则

### 4.4 写作策略配置

写作类策略至少包括：

- 默认生成目标字数区间
- 写作链是否启用“结构稿 -> 去模板化 -> 终稿”
- 是否默认启用去模板化
- 是否默认启用标题补写
- 正文节奏偏好：保守 / 平衡 / 推进型
- 风格保真优先级与剧情推进优先级

### 4.5 改写策略配置

改写类策略至少包括：

- 默认改写模式：保守 / 中度 / 激进
- 是否允许改写句式结构
- 是否允许改写段落顺序
- 是否允许增删细节
- 是否允许风格重塑
- 是否保留原桥段顺序

### 4.6 质量控制策略配置

质量控制策略至少包括：

- 阻断阈值
- `blocking / major / minor` 的打分边界
- 哪些问题类型必须阻断
- 自动修订最大轮数
- 校验失败后是否允许人工确认继续
- AI 痕迹检测敏感度

### 4.7 任务执行策略配置

任务执行类策略至少包括：

- 单任务最大重试次数
- 超时阈值
- 失败后的自动降级规则
- 是否允许中断后恢复
- 任务并发限制
- 单小说并发任务数量上限

---

## 5. 固定规则与可配置规则的边界

### 5.1 固定规则

以下规则必须写死在系统中，不允许普通配置覆盖：

- Kimi 不参与正文写作
- DeepSeek 不参与全书结构分析
- 关键结构分析低可信时不得直接进入写作
- 已锁定内容不得被自动覆盖
- 已发布版本不得被原地改写
- 写作前必须具备 `ready` 或可等价进入写作链的稳定状态

### 5.2 可配置规则

以下规则允许配置或策略覆盖：

- 默认规划模式
- 默认改写模式
- 最近章节上下文窗口大小
- 校验敏感度
- 单章分析深度
- 是否优先推进主弧或最近弧

### 5.3 半固定规则

部分规则是平台级可调，但不对普通作者直接暴露。例如：

- Prompt 模板选择策略
- 长文本分块阈值
- 某类题材的默认上下文装配模板
- 某些模型失败后的同职责降级次序

这类规则由平台或开发侧管理，不作为普通用户开关。

---

## 6. 策略档位设计

为了避免作者面对过多底层参数，系统应提供少量高层档位，底层细节由策略映射表决定。

### 6.1 写作档位

- 保守写作  
  更重一致性与已有风格延续，推进速度较慢。

- 平衡写作  
  平衡推进、表达和稳定性。

- 推进型写作  
  更强调剧情推进与章节事件量。

### 6.2 改写档位

- 保守改写  
  保持原句意与段落结构为主。

- 中度改写  
  允许重组表达和局部段落。

- 激进改写  
  允许较大幅度重塑表达，但不得违背锁定内容和核心情节。

### 6.3 校验档位

- 标准校验
- 严格校验
- 发布前校验

每个档位映射到不同的阈值、阻断规则和 issue 输出密度。

---

## 7. Prompt 与策略绑定

Prompt 不是独立漂浮的模板，它必须和策略系统绑定。

每一次模型调用至少应记录：

- `prompt_template_id`
- `prompt_version`
- `policy_profile_id`
- `policy_version`
- `context_builder_version`

当结果质量变化时，系统必须能判断变化来源：

- 是 prompt 变了
- 是策略变了
- 是上下文装配变了
- 还是模型本身返回波动

---

## 8. 配置来源与覆盖顺序

系统的有效策略由以下来源合成：

1. 固定规则
2. 平台默认策略
3. 小说级策略
4. 任务级策略
5. 系统基于运行态自动施加的安全约束

最终有效策略必须在任务启动前完成解析，并冻结为该任务的执行快照。

任务执行过程中不应读取“实时漂移的当前配置”，避免长任务前后行为不一致。

---

## 9. 面向用户暴露的配置与内部策略的边界

### 9.1 可向作者暴露的策略

作者可以理解并控制的配置包括：

- 默认规划模式
- 默认写作档位
- 默认改写档位
- 风格偏好
- 节奏偏好
- 是否允许激进改写
- 是否启用更严格的一致性校验

### 9.2 不应直接暴露给作者的内部策略

以下策略更适合内部控制：

- 长文本分块阈值
- Prompt 模板版本
- 同职责降级链
- 内部评估阈值细节
- 模型调用的上下文截断细节

作者应看到“可理解的高层行为控制”，而不是系统实现细节。

---

## 10. 配置存储与版本化要求

配置与策略需要被存储和版本化。

至少应区分：

- 平台策略版本
- 小说策略版本
- 任务执行快照

其中：

- 平台策略版本用于整体演进
- 小说策略版本用于长期个性化控制
- 任务执行快照用于回溯与复现

---

## 11. 当前重构落地要求

在当前重构中，配置与策略架构至少需要先落地以下能力：

1. 明确固定规则清单  
2. 建立平台默认策略对象  
3. 建立小说级策略对象  
4. 建立任务级策略覆盖对象  
5. 在任务执行记录中保存策略快照  
6. Prompt、策略、上下文版本统一记账  
7. 前端只暴露少量高层可理解档位
