# InkTrace 6A规则
## 全局强制
1. 严格按6阶段顺序执行，禁跳步。
2. 交付物存`docs/`，命名`[阶段]_[任务].md`。
3. 作者「孔利群」，禁Trae标识。
4. 注释/文档用中文；Python PEP8+类型注解。
5. 清洁架构：领域层、应用层、基础设施层、接口层，依赖由外向内。
6. DDD：识别聚合/实体/值对象/领域服务/仓储接口，领域独立。
7. TDD：功能前先写单元测试，核心覆盖率≥80%。
## 阶段规则
### 1. Align
- 生成ALIGN_*.md：需求边界、验收标准、疑问清单。
- 必须提问澄清，禁脑补。
- 门：边界明确，验收可测，疑问全清。
### 2. Architect
- 生成ARCH_*.md：分层图、模块、领域模型（聚合/实体/仓储接口）、技术栈。
- 门：与需求匹配，模块清晰，领域无污染。
### 3. Atomize
- 拆原子任务，生成ATOM_*.md：编号、层次、交付物、依赖、验收。
- 任务单一职责。
- 门：全覆盖需求，依赖合理，交付明确。
### 4. Approve
- 汇总前三阶段，生成APPR_*.md，提交审批。
- 门：用户明确批准。
### 5. Automate
- 按任务清单顺序执行。
- 每任务TDD：先写测试→实现→重构。同步更新代码/测试/文档。
- 遇阻塞暂停沟通。
- 门：代码规范，测试覆盖≥80%，与审批一致。
### 6. Assess
- 全量验证，生成ASSESS_*.md：需求完成、测试结果（覆盖率）、遗留问题。
- 修复至达标。
- 门：100%需求实现，全测试通过，无遗留bug。
